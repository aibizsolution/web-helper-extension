# 변경 기록

## 2026-03-13
- content script에서 shared GEO 모듈을 동적 import할 수 있도록 `modules/*.js`를 web-accessible resource로 열어, client audit 최적화 경로가 실제 확장 런타임에서도 동작하게 조정했다.
- 페이지 진단 client audit는 가능하면 현재 탭 안에서 바로 실행해 결과 객체만 전달하고, 구버전 content script 탭에서는 기존 HTML 스냅샷 방식으로 fallback 하도록 최적화했다.
- 페이지 진단 Dual Audit가 시작한 탭과 URL을 끝까지 고정해서, 검사 중 탭 전환이나 페이지 변경으로 봇/브라우저 결과가 섞이지 않도록 보강했다.
- 페이지 진단 AI 응답의 마크다운 렌더링에서 raw HTML 통과를 막고, 기존에 남아 있던 미사용 `eval` 기반 GEO 메시지 경로를 제거했다.
- 페이지 진단 탭 상단의 중복 제목/설명을 제거하고 바로 `진단 시작`부터 보이도록 정리했다.
- 페이지 진단 우선순위 로직을 페이지 유형별로 다르게 적용하고, `CSS 미디어 쿼리`처럼 영향이 작은 차이는 덜 과장되게 표현하도록 조정했다. 실행 계획은 작업 묶음 기준의 안정적인 구조로 재정리했다.
- 페이지 유형을 `일반 / 콘텐츠 / 상품 / 랜딩 / 로그인` 기준으로 판별해 일부 진단 항목을 점수에서 제외할 수 있게 했고, AI 해석은 JSON 기반 구조 응답으로 바꿔 결과를 더 짧고 일관되게 렌더링하도록 개선했다.
- 페이지 진단 AI 해석을 `현황 해석 / 우선순위 액션 / 실행 계획` 구조로 바꾸고, 낮은 점수에서 과도한 칭찬·과장 문구를 줄였다.
- 우측 탭바를 `번역 / 페이지 진단 / 검색 / 도구 / 반복관리 / 설정 / 오류 센터 / 최신버전` 구조로 재정리했다.
- `사이트`와 `텍스트`를 메인 탭에서 분리하지 않고, `번역` 탭 안의 `사이트 / 텍스트 / 기록` 서브탭으로 통합했다.
- 사용자 노출 문구의 `초고속 번역` 계열 이름을 `구글 번역`으로 정리했다.
- 선택 텍스트 드래그 액션바와 우클릭 메뉴에 검색 엔진 바로 열기 기능을 추가했다.
- `package.json`, `npm run check`, `npm run test:providers` 기준으로 개발 온보딩을 단순화했다.
- 페이지 진단 AI 해석은 번역 설정과 분리해 `OpenRouter + openai/gpt-5-mini` 고정 모델을 사용하도록 바꿨다.
- 우측 탭바의 `페이지 진단` 버튼은 사이트 모양 아이콘과 `진단` 라벨 조합으로 더 컴팩트하게 조정했다.
- `최신버전` 버튼과 provider 테스트 스크립트의 GitHub 저장소 링크를 `aibizsolution/web-helper-extension` 공개 저장소 기준으로 맞췄다.
- `오류 센터` 탭을 추가해 최근 오류/경고 로그 확인, 요약 복사, 로그 삭제를 지원한다.
- 업데이트 일자와 변경 문서 관리를 위한 운영 규칙을 AGENTS에 추가했다.
