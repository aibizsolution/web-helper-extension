# 변경 기록

## 2026-04-28 · v2.4.3
- 선택 텍스트 검색에서 확장 업데이트 직후 구버전 탭이 남아 있어도 legacy Bing 검색 요청은 실패하지 않게 처리하고, 알 수 없는 검색 엔진 값은 원인 확인이 가능하도록 engine 값과 새로고침 안내를 포함해 표시한다.

## 2026-04-15 · v2.4.2
- 검색 엔진 목록에서 Bing을 제거하고 YouTube 검색을 추가했다. 선택 텍스트 검색 팝오버, 우클릭 메뉴, 사이드패널 스마트 검색의 전체 검색 대상도 YouTube를 포함하도록 맞췄다.
- 릴리즈 생성 시 버전을 반드시 올리도록 운영 지침을 보강하고, 릴리즈 패키징 검증에서 CHANGELOG의 현재 버전 항목을 확인하게 했다.

## 2026-04-10 · v2.4.1
- `npm run release:pack` 스크립트를 추가해, 배포용 `release/` 폴더와 GitHub Releases 업로드용 zip을 한 번에 만들 수 있게 했다.
- 릴리즈 패키징 시 `manifest.json`과 `package.json` 버전 일치, `meta.js`의 `LAST_EDITED`, `README.md`의 `현재 버전 / 마지막 정리`가 서울 기준 릴리즈 상태와 맞는지 자동 검사하도록 보강했다.
- 설정 탭을 `번역 설정 / 선택 텍스트 액션바 / 사이드패널` 3개 카드로 재구성하고, 선택 텍스트 팝오버를 배경 클릭으로 닫을 수 있는 옵션을 기본 꺼짐 상태로 추가했다.
- 페이지 번역 재시도 로직을 보강해, 누락 번역 복구 응답이 세그먼트 수와 어긋나더라도 인덱스 범위를 벗어나지 않고 안전하게 마무리되도록 정리했다.
- 확장 업데이트 직후 content script 버전이 맞지 않는 탭에서는 현재 페이지를 자동 새로고침한 뒤 같은 번역 요청을 한 번 다시 시도하도록 조정했다.
- 번역 완료 히스토리의 중복 판정을 실행 단위 `runId` 기준으로 바꿔, 같은 탭에서 빠르게 연속 번역해도 각 실행이 누락되지 않도록 보강했다.
- 설정 탭 단축키 watcher를 탭 진입/이탈 수명주기에 맞춰 정리하고, 캐시 조회/삭제 메시지는 닫힌 탭·빈 응답·런타임 오류를 공통 helper에서 처리하도록 정리했다.
- `ESLint` 기반 `npm run lint`를 추가하고 `npm run check`가 문법 검사와 lint를 함께 수행하도록 기본 검증 게이트를 확장했다.
- 중간 배치 실패가 있어도 최종 상태가 단순 `번역 완료`로 보이던 문제를 줄이기 위해, 부분 실패는 `일부 실패` 상태와 상세 로그로 드러나게 조정했다.
- content script의 `WARN/ERROR` 로그도 공용 session 로그에 적재해, 배치 실패나 부분 실패가 오류 센터에 함께 쌓이도록 연결했다.

## 2026-03-26
- 선택 텍스트 추출이 `selection.toString()` 경로를 바로 타지 않도록 바꿔, 일부 페이지에서 선택 시 함께 튀던 외부 폰트/CSP 브라우저 오류를 줄였다.
- content script 전역 오류 판별을 더 보수적으로 조정해, 외부 페이지·브라우저 측 CSP 경고가 확장 내부 이슈처럼 섞여 보이지 않도록 보강했다.

## 2026-03-25
- 오류 센터가 content script의 전역 에러를 수집할 때 우리 확장 소스와 연결된 오류만 남기고, 외부 페이지나 다른 확장 프로그램의 CSP/font 경고는 기본적으로 제외하도록 정리했다.

## 2026-03-13
- 선택 텍스트 설명 팝오버의 `원문 복사 / 설명 복사` 버튼 텍스트가 살짝 아래로 보이던 정렬을 중앙 기준으로 바로잡았다.
- 설정 탭의 OpenRouter API Key 입력 아래에 현재 저장된 키 기준 `남은 크레딧 / 오늘 사용 / 누적 사용` 상태를 바로 보여주고, 우측 새로고침 버튼으로 수동 재조회할 수 있게 추가했다. 키 이름은 응답에 실제 이름 필드가 있을 때만 표시한다.
- content script에서 shared GEO 모듈을 동적 import할 수 있도록 `modules/*.js`를 web-accessible resource로 열어, client audit 최적화 경로가 실제 확장 런타임에서도 동작하게 조정했다.
- 페이지 진단 client audit는 가능하면 현재 탭 안에서 바로 실행해 결과 객체만 전달하고, 구버전 content script 탭에서는 기존 HTML 스냅샷 방식으로 fallback 하도록 최적화했다.
- 페이지 진단 Dual Audit가 시작한 탭과 URL을 끝까지 고정해서, 검사 중 탭 전환이나 페이지 변경으로 봇/브라우저 결과가 섞이지 않도록 보강했다.
- 페이지 진단 AI 응답의 마크다운 렌더링에서 raw HTML 통과를 막고, 기존에 남아 있던 미사용 `eval` 기반 GEO 메시지 경로를 제거했다.
- 페이지 진단 탭 상단의 중복 제목/설명을 제거하고 바로 `진단 시작`부터 보이도록 정리했다.
- 페이지 진단 우선순위 로직을 페이지 유형별로 다르게 적용하고, `CSS 미디어 쿼리`처럼 영향이 작은 차이는 덜 과장되게 표현하도록 조정했다. 실행 계획은 작업 묶음 기준의 안정적인 구조로 재정리했다.
- 페이지 유형을 `일반 / 콘텐츠 / 상품 / 랜딩 / 로그인` 기준으로 판별해 일부 진단 항목을 점수에서 제외할 수 있게 했고, AI 해석은 JSON 기반 구조 응답으로 바꿔 결과를 더 짧고 일관되게 렌더링하도록 개선했다.
- 페이지 진단 AI 해석을 `현황 해석 / 우선순위 액션 / 실행 계획` 구조로 바꾸고, 낮은 점수에서 과도한 칭찬·과장 문구를 줄였다.
- 우측 탭바를 `번역 / 페이지 진단 / 검색 / 도구 / 반복관리 / 설정 / 오류 센터 / 최신버전` 구조로 재정리했다.
- `사이트`와 `텍스트`를 메인 탭에서 분리하지 않고, `번역` 탭 안의 `사이트 / 텍스트 / 기록` 서브탭으로 통합했다.
- 사용자 노출 문구의 `초고속 번역` 계열 이름을 `구글 번역`으로 정리했다.
- 선택 텍스트 드래그 액션바와 우클릭 메뉴에 검색 엔진 바로 열기 기능을 추가했다.
- `package.json`, `npm run check`, `npm run test:providers` 기준으로 개발 온보딩을 단순화했다.
- 페이지 진단 AI 해석은 번역 설정과 분리해 `OpenRouter + openai/gpt-5-mini` 고정 모델을 사용하도록 바꿨다.
- 우측 탭바의 `페이지 진단` 버튼은 사이트 모양 아이콘과 `진단` 라벨 조합으로 더 컴팩트하게 조정했다.
- `최신버전` 버튼과 provider 테스트 스크립트의 GitHub 저장소 링크를 `aibizsolution/web-helper-extension` 공개 저장소 기준으로 맞췄다.
- `오류 센터` 탭을 추가해 최근 오류/경고 로그 확인, 요약 복사, 로그 삭제를 지원한다.
- 업데이트 일자와 변경 문서 관리를 위한 운영 규칙을 AGENTS에 추가했다.
